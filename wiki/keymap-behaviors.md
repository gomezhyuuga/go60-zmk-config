# Custom Keymap Behaviors

Hand-authored behaviors in `config/go60.keymap`, separate from the auto-generated
TailorKey (`*_TKZ`) home-row mods and tap-dances. These are the personal tweaks worth
remembering when re-syncing the keymap from an external editor.

## Apostrophe / sticky Cmd+Shift — `qt_swap`

Right pinky home-row key (base layer). Defined as a hold-tap:

```dts
qt_swap: qt_swap {
    compatible = "zmk,behavior-hold-tap";
    #binding-cells = <2>;
    tapping-term-ms = <190>;
    quick-tap-ms = <300>;
    flavor = "tap-preferred";
    bindings = <&sk>, <&kp>;
};
```

Used as `&qt_swap LG(LSHFT) SQT`:

| Action | Result |
|--------|--------|
| Tap | `'` (and `"` with Shift held — standard `SQT` keycode behavior) |
| Hold (>190 ms) | Sticky `LGUI+LSHFT` — the next key gets Cmd+Shift applied |

`tap-preferred` keeps apostrophe typing snappy: a quick press always emits `'`; it only
becomes the sticky modifier on a deliberate hold. The sticky key releases after the next
keypress (or the default sticky timeout). Long-hold key-repeat of `'` is no longer
possible on this key, which is rarely wanted for an apostrophe.

> History: this key was previously a `mod-morph` (`qt_swap`) that inverted the shift
> state (tap → `"`, shift → `'`). It was reverted to standard apostrophe output, then
> repurposed into the hold-tap above.

## Shift / Caps-word tap-dance — `td_shift_caps`

Both outer thumb-adjacent pinky keys on the Typing layer (`&td_shift_caps`).

```dts
bindings = <&skq LSHFT>, <&caps_word>;
```

| Action | Result |
|--------|--------|
| Single tap | Sticky Shift (`&skq`, queued) |
| Double tap | `caps_word` |

## Generic hold-tap — `tp`

Plain tap-preferred hold-tap with `&kp`/`&kp` on both sides (190 ms term). Used for keys
like `&tp LCTRL ESC` (hold → Ctrl, tap → Esc).

## Layers

Base/typing stack lives at the top of the keymap:

| Index | Name | Index | Name |
|-------|------|-------|------|
| 0 | `LAYER_Lower` | 11 | `LAYER_Cursor` |
| 1 | `LAYER_Typing` | 12 | `LAYER_Keypad` |
| 2 | `LAYER_Autoshift` | 13 | `LAYER_Symbol` |
| 3–6 | Left HRM (Pinky/Ringy/Middy/Index) | 14–17 | Mouse (Mouse/Slow/Fast/Warp) |
| 7–10 | Right HRM (Pinky/Ringy/Middy/Index) | 18 | `LAYER_Gaming` |
| | | 19 | `LAYER_Magic` |
| | | 20 | `LAYER_fn` |

Indices 3–10 are per-finger home-row-mod implementation layers; 14–17 are the mouse
stack; 18 is gaming. For a visual, browsable view of the human-facing layers see
[layout viewer](layout.html) (generated by `make layout-html`).
